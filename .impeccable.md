## Design Context

### Users

Developers and teams using git-agent (AI-first Git CLI) for conventional commits and atomic splits. Primary context: marketing home and command documentation on the web.

### Brand Personality

Technical, precise, and trustworthy — tool-like clarity without corporate blandness. The UI leans terminal-adjacent (monospace accents) while remaining readable for scanning and comparison tables.

### Aesthetic Direction

Dark foundation (`#050505`), warm paper cards (`#f0ede6`), high contrast ink. Prefer intentional hierarchy (sans for reading, mono for labels and display rhythm) over decorative chrome.

### Design Principles

1. **Readable first** — body text at least 1rem; avoid illegible micro type except non-critical metadata.
2. **Plex cohesion** — IBM Plex Sans for prose and UI sans; IBM Plex Mono for system-like labels and hero display.
3. **Predictable scale** — rem-based tokens and a modest modular step between levels; fluid clamps only where headings need to breathe across viewports.
